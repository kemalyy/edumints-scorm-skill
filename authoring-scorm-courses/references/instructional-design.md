# Instructional design

**Objective first.** One measurable objective per micro-course, with a Bloom verb (identify, apply,
analyze…) — not "understand". Everything maps to it; cut what doesn't.

**Structure (the spine):**
1. Hook / title — relevance, what's in it for the learner, duration.
2. Content chunks — one idea each, ≤2 screens, mixed media. Microlearning: 3–8 min total.
3. Practice — low-stakes interaction after every 1–2 chunks (retrieval beats re-reading).
4. Assessment — aligned to the objective; feedback on correct AND incorrect.
5. Summary — recap + score/completion + next step.

**Anti template-fatigue (the #1 learner complaint about cloud authoring tools):** vary screen types. Don't ship
8 identical content slides. Alternate content ↔ interaction ↔ scenario ↔ animation. Use accordions/tabs to
chunk dense info, flashcards for terms, branching for decisions, timeline for sequences.

**Engagement principles:** active > passive; concrete examples + scenarios > abstract rules; address the
learner ("you"); show consequences (branching); keep cognitive load low (one idea per screen).

**Accessibility is non-negotiable:** plain language, alt text, captions, sufficient contrast (themes handle
contrast), keyboard-operable (all built-in types are). High-contrast theme for a11y-critical audiences.
